## Описание
Проект состоит из двух сервисов
DocumentService -- сервис для работы с документами. Сохраняет, переводит между статусами, создает активити фиды и записи в реестр утрверждений.
UtilService -- сервис в котором реализован многопоточный аппрув, массовое создание документов, фоновые задачи по сабмиту и апруву. По сети ходит в DocumentService и выполняет задачи.
Собирается в docker-compose, в одной сети.
[Подробное описание проекта](Подробное_описание_по_ТЗ.md)

Дополнительно поднимается контейнер с базой Postgres, контейнер с promtail для сбора логов контейнеров, loki для аггрегации, grafarna для просмотра логов. 

### Как запустить
#### В идеале
Все что нужно сделать, это сдернуть проект, в корне проекта собрать docker-compose. 

Нужна доступность портов 
- 8080(document-service),
- 9090(util-service),
- 3000(loki),
- 15432(postgres),
- 3100(grafarna).

Нужен доступ в сеть для docker-compose, чтобы подтянуть исходные образы. 

#### Если compose не стартанул
Нужно создать базу Postgres локально, данные порта-имени базы-имени юзера лежат в application-local.yml в модуле DocumentService. Подтянуть зависимости, запустить DocumentServiceApplication, запустить UserSerivceApplication. Логи будут писаться в поток вывода, файловое логирование не настроено. Предполагаю, что сервисы запустятся из Intellj IDEA, в ее выводе можно спокойно искать по содержимому.

### Где смотреть
#### В случае нормально стартовавшего docker-compose
http://localhost:8080/swagger-ui/index.html -- апи document service

http://localhost:9090/swagger-ui/index.html -- апи util service

http://localhost:3000 -- Grafarna(admin admin), в ней логи сервисов(ссылка на буквы про логи).

Если нужно в базу: Каким-нибудь сервисом администрирования баз данных(в моем случае PgAdmin) подключаемся к localhost:15432, креды в docker-compose.
#### В ином случае
Логи в выводе IDE, сам поиск по логам остается таким же.
API через Postman. Коллекция запросов для постмана лежит в проекте в директории postmanCollection, есть 2 и 2.1 версия экспорта из постмана.
